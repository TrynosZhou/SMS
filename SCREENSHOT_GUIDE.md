# Screenshot Guide for User Manual

This document provides guidance on which screenshots to capture for the User Manual.

## Screenshot Requirements

### General Guidelines

1. **Resolution**: Capture screenshots at 1920x1080 or higher resolution
2. **Format**: Save as PNG or JPG
3. **File Naming**: Use descriptive names (e.g., `login-page.png`, `student-list.png`)
4. **Annotations**: Add arrows, highlights, or callouts to point out important features
5. **Privacy**: Blur or remove sensitive information (student names, IDs, etc.) if needed

## Screenshot Annotation Guidelines

### When to Add Annotations

1. **Highlight Important Fields**: Use arrows or highlights to point out key input fields
2. **Show Workflow**: Use numbered steps to show process flow
3. **Indicate Actions**: Use callouts to show where to click
4. **Explain Features**: Add text boxes to explain complex features

### Tools for Annotations

- **Windows**: Snipping Tool, Snip & Sketch, or third-party tools like Greenshot
- **Mac**: Screenshot tool with Markup, or Skitch
- **Online**: Annotate images using tools like Annotely or Photopea

### Annotation Best Practices

1. **Keep it Simple**: Don't overcrowd screenshots with annotations
2. **Use Consistent Style**: Use the same color scheme and arrow style throughout
3. **Clear Text**: Ensure text in annotations is readable
4. **Highlight Key Areas**: Only highlight the most important elements

## Privacy Considerations

### What to Blur/Remove

1. **Real Student Names**: Replace with "Student Name" or "John Doe"
2. **Real Student IDs**: Replace with "STU001", "STU002", etc.
3. **Real Contact Numbers**: Replace with "07XXXXXXXX" or "0771234567"
4. **Real Email Addresses**: Replace with "example@email.com"
5. **Real Addresses**: Replace with "123 School Street, City"
6. **Real Financial Data**: Use sample amounts (e.g., $100, $500)

### Tools for Privacy

- **Blur Tool**: Use image editing software to blur sensitive information
- **Redaction**: Use black bars or white boxes to cover sensitive data
- **Mock Data**: Use sample/test data instead of real data

## Quality Checklist

Before including screenshots in the manual, verify:

- [ ] Screenshot is clear and readable
- [ ] All text is legible
- [ ] Important elements are visible
- [ ] No sensitive information is exposed
- [ ] File size is reasonable (< 500KB per image)
- [ ] File name follows convention
- [ ] Screenshot matches the section content
- [ ] Annotations (if any) are clear and helpful
